# C7-001 — Simple XGBoost vs elaborate pipeline (operator challenge, P3/P4)

Program: P3/P4 · State: CLOSED

## Hypothesis (operator)
Raw data + a few simple features + XGBoost produces an edge at least as
good as the elaborate 205-feature LightGBM pipeline — i.e. the pipeline
is over-engineered.

## Method
Head-to-head, identical folds (walk-forward, expanding, annual, purge
96), identical G2 barrier label (y_tp_long), 2014-2026, out-of-sample.
Eight causal features from raw OHLC (ret1/4/16, vol16, mom16,
dist_sma20, range, hour) vs the full 205. XGBoost and LightGBM both run
on the simple set to separate "model" from "features".
Script `research/cycle7/simple_xgb.py`.

## Results (`research/cycle7/results/c7_simple_xgb.json`)
| Model | Feats | Mean OOS AUC | Yrs>0.5 |
|---|---|---|---|
| simple-8 + XGBoost | 8 | 0.5086 | 11/13 |
| simple-8 + LightGBM | 8 | 0.5067 | 11/13 |
| full-205 + LightGBM | 205 | **0.5239** | 13/13 |

## Verdict
REFUTED. The simple approach is **weaker**, not equal: 0.5086 vs 0.5239
mean AUC, 11/13 vs 13/13 positive years. The 197 extra features add real,
consistent out-of-sample signal. XGBoost ≈ LightGBM on the same features
(0.5086 vs 0.5067) — the model choice is immaterial; feature richness is
what moves AUC. Confirms C1-MODEL (model family doesn't matter here) and
C1-FEAT (many engineered features carry OOS value) from a new angle.

## Lesson (the load-bearing one)
Even the *winning* model here (0.5239) does not survive 2.5bp costs
(C1-ECON breakeven ≈1.3bp). The simple-XGBoost model at 0.5086 is a coin
with a whisper and cannot produce a cost-positive strategy on honest
walk-forward. So a "simple XGBoost strategy that works on 3 years" is
almost certainly explained by evaluation, not signal: no cost subtracted,
in-sample/random split, or a single favourable volatility regime
(cf. C5 sliding intervals: 2015-2019 was strongly positive for
everything, 2022-2026 negative for everything). The instrument's edge is
real, tiny, cost-fragile, and regime-concentrated regardless of feature
count or model library.
