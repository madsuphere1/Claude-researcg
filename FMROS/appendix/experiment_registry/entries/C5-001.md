# C5-001 — Window × interval × metric mapping study (P10, exploratory)

Program: P10 · State: PRE-REGISTERED (exploratory mapping, not confirmatory)

## Question
Across every training-window and evaluation-interval grouping, compute a
full quant-metric suite (28 metrics), then determine **which metrics
carry forward-predictive information and which are decorative**. Operator
framing: "not just a dump — understand the pattern, and which metrics
actually help and which don't."

## Why this is exploratory, and what that constrains
This is a MAPPING study over a grid of groupings; it has no single
accept/reject decision rule because it is not testing one hypothesis —
it is characterising a surface. Per Constitution, exploratory work may
generate hypotheses but confirms nothing; any metric that looks
predictive here becomes a *pre-registered* hypothesis for new data
(C3-008 territory), not a validated finding. Small-sample caveat is
declared up front: 13 test years yield ~11 overlapping trailing→forward
pairs per policy; pooled across 5 policies ~55 pairs. Correlations are
indicative, not significant-by-default.

## Grid (declared)
* Training windows (how the model is fit): expanding, rolling-5,
  rolling-4, rolling-3, rolling-2. (rolling-4/-3 retrained for this
  study; the rest cached.)
* Evaluation intervals (what slice performance is measured over): the
  full history, plus sliding calendar spans of 2, 3, and 5 years
  stepped by 1 year across 2014-2026.
* Metric suite (28, frozen list in `research/cycle5/metrics.py`):
  n_trades, trades_per_year, expectancy_R, median_R, total_R, win_rate,
  avg_win_R, avg_loss_R, payoff_ratio, profit_factor, std_R, sharpe,
  sortino, downside_dev, max_dd_R, calmar, recovery_factor, ulcer_index,
  skew, kurtosis, var95, cvar95, tail_ratio, omega, max_consec_wins,
  max_consec_losses, best_R, worst_R.

## Analysis plan (declared before running)
1. **Descriptive**: full-history metric suite per training window; the
   sliding-interval expectancy (+ key metrics) matrix per window.
2. **Informativeness** — the operator's core question. For each metric,
   over all (trailing-interval → next-year expectancy) pairs pooled
   across policies:
   * forward Spearman |ρ| with next-year realized expectancy (does the
     metric *predict*?);
   * persistence: lag-1 autocorrelation of the metric across intervals
     (is it stable enough to act on?);
   * discrimination: does the metric separate the profitable policy
     (expanding) from the dead one (rolling-2)?
   Rank metrics; classify each as HELPS (forward |ρ|>0.3 and persistent),
   WEAK, or DECORATIVE (forward |ρ|≈0). Expected honest result, stated
   in advance: most metrics will NOT predict forward performance here,
   consistent with the program's regime-concentration / low-persistence
   findings — a null-leaning result is the likely and acceptable outcome.

## Results (run 2026-07-10)
Streams: expanding 748, rolling-5 598, rolling-4 807, rolling-3 602,
rolling-2 628. Full metric suite, sliding 2/3/5-yr interval grids, and
informativeness panel (42 trailing→forward pairs) in
`research/cycle5/results/`. See `report/CYCLE5.md`.

Key findings:
* Full history: only expanding is profitable (+0.106R); rolling ordering
  is NOT monotone (roll-4 −0.022 > roll-5 −0.037) — sub-expanding
  differences are inside the noise. The clean monotone signal is risk
  (max-DD −29R expanding → −50R+ rolling), not return.
* Sliding intervals: edge decayed for EVERY window; 2022-2026 negative
  across all five; rolling never rescues the recent period at any length
  (2024-2026: expanding −0.056 best, rolling-2 −0.285 worst). Recent
  drag is a regime/time problem, not a window problem.
* Informativeness (the operator's core question): return metrics
  (expectancy/PF/Sharpe/Sortino/omega) discriminate perfectly
  (ρ_disc=+1.0) but predict forward barely (ρ_fwd≈+0.14) — past
  performance is not a forward signal. Only RISK/TAIL metrics predict:
  HELPS = {max_dd_R (ρ_fwd+0.34, persist 0.51, disc 0.90),
  max_consec_losses (−0.35), tail_ratio (+0.31)}. Losing-streak informs,
  winning-streak decorative. DECORATIVE = {median_R, avg_win_R,
  payoff_ratio, downside_dev, max_consec_wins}; several are structurally
  constant across windows (barrier-geometry-pinned) so blind by design.

## Verdict
CLOSED (exploratory mapping). Generated three pre-registration
candidates for C3-008's 2027+ watch-list (drawdown / consec-losses /
tail-ratio as forward predictors). 42-pair / 23-metric multiplicity
means HELPS metrics are hypotheses, not verdicts — as the plan declared.
