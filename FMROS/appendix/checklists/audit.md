# Auditor Checklist (per cycle)

- [ ] Sample ≥ 2 registry verdicts; regenerate their numbers from scripts on a clean state
- [ ] Every number in the cycle report traces to a committed results file
- [ ] Graveyard check: failures reported at same prominence as wins
- [ ] Exploratory vs confirmatory labeling correct everywhere (X1-style caveats present)
- [ ] Grep git history of experiment scripts: any parameter changed between pre-registration commit and run commit? (= void)
- [ ] Blocking levels: was the coarser block computed wherever profit concentration is visible? Did it gate?
- [ ] Multiplicity: every screened universe declared; deflation applied
- [ ] Anchor set current (re-run if simulator/features/data changed; deltas within tolerance)
- [ ] Data changelog (A5) reconciled with cache invalidations
- [ ] Estimated vs realized information gain recorded for Chief Scientist calibration
- [ ] PASS/FAIL recorded in the cycle report, addressed to the operator
